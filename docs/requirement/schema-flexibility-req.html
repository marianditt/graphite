REQ-2 Schema Flexibility

Statement

WHEN a project requires custom knowledge structures, THE system SHALL allow users to define node kinds and edge types in graph.schema.

Addressed by ARC-2 Schema-Driven Graph Architecture.

Rationale

Different projects need different knowledge models. A fixed set of kinds limits Graphite's applicability.

Acceptance Criteria

Scenario: Custom kind definition

GIVEN a project with a custom component kind defined in graph.schema WHEN graphite validate graph is run THEN all component nodes SHALL be accepted.

Scenario: Custom edge definition

GIVEN a custom depends_on edge from component to component defined in graph.schema WHEN graph validation runs THEN component nodes with depends_on edges SHALL pass validation.

Scenario: Unknown kind rejection

GIVEN a graph node with kind undefined_kind not declared in graph.schema WHEN graph validation runs THEN the system SHALL emit a schema-conformance diagnostic.

Scenario: Edge target existence

GIVEN a node with an edge declaration addressed_by: [nonexistent-node] WHEN graphite validate graph is run THEN the system SHALL emit a diagnostic with rule broken-edge-target AND the diagnostic SHALL include:

  • detail: which node declared the edge, which target is missing, and that the target ID does not correspond to any known node
  • fix: how to either create the missing node or correct the target ID
  • hint: a general principle that every edge target MUST resolve to an existing node ID in the graph
