Compiler Pipeline

The compiler pipeline has four stages:

  1. Parse: Read .node files into a Graph via the schema and node parsers
  2. Validate: Run structural checks (reachability, tree constraint, cycles, schema conformance, body-edge usage, anchor resolution)
  3. Resolve: Match evidence anchors to source file locations via the scanner and sidecar resolvers
  4. Render: Generate static HTML with heading depth derivation and hyperlinked edge references

See SVC-002 for the implementation. The stages are designed per ADR-004 (narrative consistency), ADR-005 (heading derivation), ADR-002 (evidence resolution), and ADR-008 (type system).
