ADR-8 Schema Design

The type system consists of named kinds (requirement, adr, service, test) each with a short display key (REQ, ADR, SVC, TST), and named edges with from/to kind constraints. Built-in kinds index, evidence, and any are always available. Built-in edge contains is reserved for index nodes.

Every edge is validated against the schema at compile time — an ADR cannot implemented_by a service because that edge is defined as requirement → service. See SVC-2 Compiler for implementation. The stable identity design is at ADR-9 Stable Identity and the index pattern at ADR-6 Index Pattern.
